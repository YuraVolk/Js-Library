Toast

The toast component which represents a panel that smoothly transitions in and then smoothly transitions out is commonly used and has many applications: notifications, cookie requests, and et cetera. This is a very simplistic, and just gives the necessary attributes to quickly setup a smooth animation. The basic configuration is to insert all required content inside the component. This component accepts the following props:

  • isOpen, specifies whether the toast is open.
  • autoCloseDelay, a feature by default disabled that automatically closes the toast after the milliseconds passed to this attribute.
  • transitionDuration, that specifies the duration of the transition.

In order to correctly configure the animation and make sure it is easy to extend, the toast class wraps around a transition class. It applies the following set of classes:

  • enter-active, when the appearing animation is about to start.
  • enter, when the appearing animation has started.
  • enter-done, when the appearing animation has finished.
  • leave-active, when the leaving animation is about to start.
  • leave, when the leaving animation has started.
  • leave-done, when the leaving animation has finished.

You need to query the element, preferably through a ref, and extract the provided transitionDirective and apply it to the class attribute of the element you want to animate: the transition will automatically get in sync through its own semaphore-based asynchronous data management, making sure that only on data change the transitions will get started.
